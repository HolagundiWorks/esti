AORMS is a **browser product**. Staff work in **AORMS-Studio** (and, when launched,
**AORMS-Consultancy**) from a modern desktop browser. There is no native desktop
shell to download, no offline Manager app, and no Lite/Pro packaging split.

## Why we dropped desktop

Desktop installers fragmented the story: different builds, update paths, and
licensing language for the same operational spine. Architecture consultancies
already live in the browser for drawings, mail, and client portals. Putting the
office record behind another installer did not make fee recovery or revision
control clearer — it only added a second surface to maintain.

So the product direction is explicit:

- **Web-only** — no Tauri/Electron shells, no `/download` installers
- **One workspace URL** — `studio.aorms.in` for architecture staff; platform home at `aorms.in`
- **One Standard licence** — storage and AI usage, not tiered desktop SKUs

## What that means for practices

You sign in, open the project, and the fee stage, drawing issue, client approval,
and site note share one record. ESTI and Studio Intelligence run against that
record in the same session. Client, consultant, and contractor portals stay on
the web — the same model as the staff workspace.

If an older campaign mentioned a desktop app, Lite/Pro, or offline Manager,
treat it as superseded. The live path is **sign up / demo → browser workspace**.
