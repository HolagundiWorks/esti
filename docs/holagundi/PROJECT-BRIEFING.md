## Compiling the design brief

After the client returns the questionnaire, the lead architect produces a **Project Design Brief** — a concise, authoritative document the studio uses for Concept Design. The questionnaire is raw input; the brief is curated narrative and tables.

### Brief structure (recommended)

| Section | Source (questionnaire) | Content |
| --- | --- | --- |
| **1. Project summary** | §2 Project information | Nature of project, intended use, BUA, phasing, budget, finance, start date |
| **2. Client and occupants** | §1 Basic info, §3 Occupants | Contact, household table, staff requirements |
| **3. Site context** | §1 Site, terrain, orientation | Addresses, plot size, topography, vegetation, soil, natural features, orientation measurements |
| **4. Design drivers** | §4–§9 Preferences | Orientation, doors, views, basement, Vastu, style, loved places, activities, indoor/outdoor |
| **5. Accommodation schedule** | §10 Space requirements | Room list with area, floor, description |
| **6. Room character** | Room templates | Per-room ambience, finishes, furniture, connectivity |
| **7. Materials palette** | §11 Material preferences | Construction, flooring, walls, cabinetry, seating, beds |
| **8. Assumptions and exclusions** | Staff notes | Survey pending, authority constraints, items deferred to Concept |
| **9. Approval** | — | Client sign-off date and name |

### ESTI mapping

| Brief section | Where to record in ESTI |
| --- | --- |
| §1 Basic information, §2 Project information, §3 Occupants | Project **Info** tab — questionnaire sections |
| §4–§11 Design drivers, accommodation, materials, room detail | Project **Info** tab — accordion sections |
| §8 Assumptions, §9 Client approval | Project **Info** tab — Assumptions & approval |
| Compliance summary (permits) | Project **Info** tab — Compliance/permits section |
| Current design stage | Project **Info** (read-only tag) + **Settings** → Project stages |
| Executive summary | Project **Appointment → scope summary**; proposal **scope** field |
| Full brief narrative export | Project **Info** → compiled brief (markdown) or issued letter in **Documents** |
| Site and context | Project **Info** — site & context section |
| Accommodation schedule (tasks) | **Tasks** under Initiation & Brief; **Info** accommodation table |
| Style and loved places | **Mood board** titles; **Info** design drivers |
| Material preferences | **Info** materials palette; **Specification sheet** rows |
| Room-level detail | **Info** room character; mood boards + spec sheets per room |
| Client approval (signed) | **Info** approval note; **Approval** record or signed PDF |

### Quality gate before Concept Design

The brief is **ready** when:

- Budget and BUA are stated or explicitly marked TBD with a decision date
- Every bedroom and wet area in the program appears in §10 or is marked N/A
- Kitchen type and open/closed preference are decided
- Vastu requirement is yes, no, or consultant engaged
- At least entrance, living, kitchen, and master bedroom room templates are complete
- Principal has reviewed and client has approved
